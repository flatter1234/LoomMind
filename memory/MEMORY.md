# 记忆手册

长期约定、术语与操作说明；**主要由人工维护**。启动会话时，系统提示会附带本文件的节选（有长度上限），完整内容始终以本文件为准。

## 本仓库 `memory/` 目录分工

| 文件 | 作用 |
|------|------|
| `MEMORY.md`（本文件） | 手册与约定；经 `read_handbook` 节选注入系统提示。 |
| `memory_summary.md` | **compass 会话压缩**成功时，由 `src/memory` 自动追加「较早对话摘要」，供跨会话复用。 |
| `planning_long_term.md` | **规划模块**长期条目；行为 `- [时间戳] 要点` 列表，由 `src/planning/memory.py` 读写。 |

## 术语

- **compass 摘要**：压缩后写入 `memory_summary.md` 的短文本，带时间戳分段。
- **规划长期记忆**：`planning_long_term.md` 中带时间戳的列表行，用于规划场景回看最近若干条。
- **手册节选**：`MEMORY.md` 注入提示时的截断版本；重要约定请写在前部以便优先被节选。
- **可编辑块**：Markdown 中自 `### 标题` 至行 `/END` 的区间，表示一段可整体替换的「当前态」；可选用 `[ALWAYS_LOAD]`、`[MERGE_LOAD]`、`[KEYWORD_LOAD]`、`[RECENT_LOAD]` 等标记加载优先级（与外部检索/编排协同时使用）。
- **记忆强度 `[Strength: Low/Medium/High]`**：人工或工具按编辑频率、重要性打的标签，便于排序与裁剪（概念约定，非代码强制字段）。

## 记忆模型与编写原则（概念）

在向量库之外，本仓库鼓励 **Git + Markdown** 友好、可 blame、可 diff 的写法：

- **可编辑的当前态**：优先原地更新，控制单文件体量（建议约 2000 行内），避免无限堆叠历史正文。
- **记忆分化（概念）**：传记式（人物/自我认同）、情节式（时间线）、事实/语义式（主题知识）可分文件或分块维护；跨文件用相对路径 Markdown 链接。
- **有界块**：用 `/END` 闭合块，便于检索与局部加载；可标注事件边界、时间漂移说明等。
- **Git**：需要追溯时可用 `git log -p`、`git diff`、`git blame` 等；与本仓库自动化无关时亦可仅在理念层参考。

## 可选：扩展目录布局（若单独用 Markdown 树托管人物/时间线）

若将来在**独立目录**中维护更细粒度记忆（非当前 `memory/` 三文件），可采用扁平结构示例：

- `memories/people/*.md`：人物/自述侧写  
- `memories/contexts/*.md`：主题事实  
- `timeline/YYYY-MM.md`：按月时间线  
- `index.md` / `episodes_index.md`：索引（可手工或工具生成）  

具体模板与链接约定见上文「可编辑块」与「记忆分化」；**与 LoomMind 核心代码无硬绑定**，按需采用。

## 会话到记忆的写入管线（概念参考）

自动化或半自动化将对话沉淀为文件时，典型阶段包括（实现可分散在 compass、规划模块或未来写入器中）：

1. 加载核心上下文（如各文件中 `[ALWAYS_LOAD]` 块拼接）。  
2. 识别实体/主题：新建文件或定位待更新块。  
3. 基于语义上下文的搜索替换更新正文（避免纯行号）。  
4. 更新时间线或摘要类文件；重建或增量更新索引类文件。  
5. 变更留在工作区直至**显式提交**（若使用 Git），以保证可审阅、可回滚。

相关字段概念（与外部实现互通时）：`session_transcript`、`staged_changes`、`semantic_index`、`memory_strength`、`master_index` 等。
